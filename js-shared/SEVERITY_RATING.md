# 前端 JS 审计统一分级标准

所有 JS 审计 Skill 共享此分级体系。评级以“前端证据 + 可验证后端影响”为核心，避免把纯客户端 UI 绕过误报为高危。

## 0. 噪音控制与评级封顶

以下问题在前端 JS 白盒审计中信噪比较低，默认不进入“高危/严重漏洞”列表：

| 类型 | 默认处理 | 评级封顶 |
|------|----------|----------|
| CORS 配置宽松/缺失 | 优先作为后端 API 待验证项或环境加固项；没有可复现跨域读取敏感数据链路时不报漏洞 | 最高 M；不得 H/C |
| 点击劫持、缺失 `X-Frame-Options`、缺失 `frame-ancestors` | 默认加固项；没有明确敏感操作被 iframe 诱导完成的 PoC 时不报漏洞 | 常规最高 L；有完整敏感操作 PoC 最高 M；不得 H/C |
| 缺失/宽松 CSP | 默认加固项；若存在 XSS，按 XSS 链路评级，不按 CSP 缺失评级 | 最高 L；不得 H/C |
| Mixed content、通用安全响应头缺失 | 默认信息项/加固项 | 最高 L；不得 H/C |

如果 CORS/点击劫持只是帮助触发了真实 API 越权、敏感信息泄露或账号影响，应按真实根因（`API`、`SECRET`、`XSS` 等）评级，而不是把 CORS/点击劫持本身标为高危。

## 1. 计算公式

```
Score = R × 0.40 + I × 0.35 + C × 0.25
CVSS 映射 = Score / 3.0 × 10.0
```

| 维度 | 说明 |
|------|------|
| R Reachability | 外部攻击者或低权限用户是否能直接触达该代码路径、接口或敏感信息 |
| I Impact | 对账号、数据、权限、后台接口、业务操作或供应链的实际影响 |
| C Complexity | 利用是否依赖复杂前置条件；越容易利用分越高 |

## 2. 评分表

### 可达性 R

| 分值 | 判定标准 |
|------|----------|
| 3 | 无需登录即可加载 JS、访问路由或直接请求 API |
| 2 | 需要普通账号或任意登录态 |
| 1 | 需要特定角色、内网、灰度环境或构建产物泄露 |
| 0 | 死代码、无法触达、测试假数据且无运行引用 |

### 影响范围 I

| 分值 | 判定标准 |
|------|----------|
| 3 | 后端未授权关键操作、管理员接口暴露、有效云密钥、账号接管、批量敏感数据泄露 |
| 2 | 普通越权、有效业务 Token/内部接口泄露、可利用 DOM XSS、敏感配置泄露 |
| 1 | 低敏信息、调试端点、有限客户端绕过、需配合其他漏洞 |
| 0 | 无实际影响或仅 UI 显示差异 |

### 利用复杂度 C

| 分值 | 判定标准 |
|------|----------|
| 3 | 单次请求、修改 localStorage/URL/hash 即可触发 |
| 2 | 需要构造参数、低权限账号、组合 source map 或反混淆 |
| 1 | 需要特定租户、时序、复杂交互或受害者点击 |
| 0 | 有效防护，无法利用 |

## 3. 等级映射

| 前缀 | Score 区间 | CVSS | 图标 | 含义 |
|------|------------|------|------|------|
| C | 2.70-3.00 | 9.0-10.0 | 🔴 | 可导致系统沦陷或关键数据完全暴露 |
| H | 2.10-2.69 | 7.0-8.9 | 🟠 | 可造成重大损害或关键后台功能越权 |
| M | 1.20-2.09 | 4.0-6.9 | 🟡 | 可造成一定损害，需要前置条件 |
| L | 0.10-1.19 | 0.1-3.9 | 🔵 | 加固项或有限信息泄露 |

## 4. 漏洞编号

```
{等级}-{类型}-{序号}
```

| 类型 | 说明 |
|------|------|
| SECRET | 硬编码密钥、Token、账号、内部配置泄露 |
| ROUTE | 前端路由守卫、登录状态、角色判断绕过 |
| API | 未授权 API、越权 API、参数可控接口 |
| XSS | DOM XSS、模板注入、HTML 注入 |
| REDIRECT | 开放跳转、OAuth 回调污染 |
| MSG | postMessage、BroadcastChannel、WebSocket 信任边界缺陷 |
| POLLUTION | 原型污染、客户端对象合并污染 |
| STORAGE | 本地存储、Cookie、JWT、Service Worker 缓存风险 |
| SUPPLY | npm 依赖、source map、构建链泄露 |
| CONFIG | 低优先级安全头、CORS、点击劫持、CSP 等加固项；默认不进入高危列表 |
